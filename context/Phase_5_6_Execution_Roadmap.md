# DNaIR-LLM 项目收官阶段执行路线图 (Phase 5 - 6)

本文档基于顶级信息检索与数据挖掘学术会议（如 SIGIR, KDD, WWW）的苛刻审稿标准，为项目的最终实验决战与论文撰写阶段制定了最严密的工程发文路线图。

本路线图的核心指导思想是：**“吃透小数据集的秒级迭代红利，稳拿大数据集的泛化证明，并在论文图表中进行极致的学术算力分配。”**

---

## 🛑 学术逻辑避坑指南：致命的“超参数泛化陷阱”

在过往的许多研究中，极易犯下一个致命的工程错误：在小型密集数据集（如 ML-100K）上进行网格搜索（Grid Search），找到了一组“完美”的超参数（如对比损失权重 $\lambda_{cl}$、探索率 $\epsilon$、LLM 质量分权重 $\beta$ 等），然后直接带着这组参数去跑极度稀疏或规模庞大的目标数据集（如 FilmTrust 或 ML-1M）。

**惨痛后果**：强化学习（RL）和对比学习（InfoNCE）对超参数存在极高的环境敏感依赖。
- 在 ML-100K 上调出的“最优参数”往往是局部过拟合（Local Optima）。
- 若将同一套参数生搬硬套到极度稀疏的系统（如 FilmTrust）中，由于 ID 协同矩阵近乎失效，模型极有可能直接崩溃（Loss 无法收敛），或表现劣于 Baseline。

**破局原则**：**每一个核心数据集，都必须拥有自己独立的超参数网格搜索生命周期。**

## 🗺️ 终极落地执行路线 (The Final Playbook)

为了兼顾开发效率与顶级学术说服力，后续所有工作将严格恪守以下四个战略阶段推进：

### 🎯 Phase A：在 ML-100K 上“排雷与练兵”（预演决战）
**目标：在低试错成本的环境中，完成所有的外围建设与初步概念验证（Proof of Concept）。**

1. **引敌入局（Baseline 集成）**：在 ML-100K 环境下接入并跑通所有的 Baseline 模型代码，确保输入输出对齐。选取矩阵必须涵盖以下四大类（10种顶尖或经典模型）：
   - **传统与静态模型 (ItemCF, DIN)**：揭示非交互式、纯监督学习模型在动态环境与长尾数据挖掘中的根本性劣势。
   - **经典强化学习与公平性模型 (DNaIR, SAC4IR, FCPO)**：确立未引入大语言模型语义特征的强化学习性能天花板，验证语义增强的边际贡献。
   - **最新 LLM 交互式推荐「2024-2026」 (LERL, LAAC, LLM-IPP)**：与同样利用大语言模型进行分层规划或对抗生成的顶级前沿算法直接竞争，彰显本架构的先进性。
   - **语义融合与对比学习框架 (CSRec, CLLMRec)**：在特定模块层面，比较互信息最大化方案与其他知识图谱或视图增强策略在解决数据稀疏性上的效率差异。
2. **跑通全链路机制**：确保无论是我们的双塔 DQN 还是 Baseline，在 ML-100K 上都能无 Bug 跑通我们严苛的多维（4D）评价体系，并正常输出 `MRMC`, `LtC`, `Recall_Cold` 等指标。
3. **初调与特征可视化（关键加分项）**：
   - 在 ML-100K 上做一个初步的超参数搜索，证明“LLM + RL”能够打败 Baseline。
   - **执行流形可视化**：利用 t-SNE / PCA 降维技术，将高维空间的特征向量画成散点簇。用视觉图表直接震撼审稿人：**原本在纯 ID 空间中孤立的冷门商品，在加入了 InfoNCE 和 LLM 语义后，是如何被成功拉入热门用户意图簇中的。**（在 100K 上画图最快、最清晰，防止百万级数据密恐糊图）。

### 🧊 Phase B：数据扩容与弹药准备（随时异步执行）
**目标：准备“一大一小，一密一稀”的核心弹药库，满足顶会的鲁棒性考核。**

1. **规模担当 (ML-1M)**：下载 MovieLens 1M（百万级交互）。由于我们已经打通了处理管道，只需针对性调用 DeepSeek API 获取剧情文本，并用 BGE 生成 `.npy` 语义矩阵。
2. **极度稀疏性担当 (FilmTrust)**：该数据集密度仅为 1.14%，是纯 ID 协同过滤的“坟墓”。抓取其电影实体的元数据并转化为向量。在这里，LLM 语义将产生绝对的降维打击！

### ⚔️ Phase C：决战新战场（主表数据产出）
**目标：在目标数据集上榨干模型潜力，产出毫无破绽的 T-Test 验证数据。**

1. **重新洗牌调优 (Re-tuning)**：在 ML-1M 和 FilmTrust 上，**必须重新进行网格搜索（Grid Search）**！
   - *FilmTrust 调优提示*：极度稀疏，需**大幅调大** LLM 语义奖励的权重 $\beta$ 和 InfoNCE 的权重 $\lambda_{cl}$，强迫模型依赖语义先验知识。
   - *ML-1M 调优提示*：数据量大，需**调小**学习率，并增加 Replay Buffer 的容量，调整批量大小。
2. **无情碾压与显著性检验**：
   - 用各自在验证集上调好的“最强完全体参数”，去压制被“降维打击”的 Baseline。
   - 彻底执行**多次随机种子跑分（5 Random Seeds）**，计算平均值，并通过 Wilcoxon 或 Paired t-test 检验，为领先的成绩打上令人信服的 `**` (p<0.01) 星号。

### 📄 Phase D：论文图表布局与叙事策略（Storylining）
**目标：极限压榨计算性价比，将好钢用在刀刃上。合理分配数据集在论文各章节的情态展位。**

- **主实验表格 (Main Results Table)**：只展示 **ML-1M** 和 **FilmTrust**。向审稿人证明框架在【千万级商业系统可扩展性】与【冷启动不毛之地鲁棒性】上的绝对统治力。
- **深度消融实验 (Ablation Study)**：放在 **ML-1M** 上做。通过“拔掉 InfoNCE”、“拔掉 DSAG 语义生成”、“拔掉 LLM 语义奖励”，展现哪怕缺失任何一环，大盘性能都会惨烈下跌。
- **超参数敏感性分析 (Hyperparameter Sensitivity) & t-SNE 可视化**：堂而皇之地放在 **ML-100K** 上做！在论文中写明：*“为了更清晰、颗粒度更细地进行超参数曲面刻画和特征空间可是化分析，本小节在 ML-100K 上深入剖析...”*（大幅降低百万级数据集超参绘图耗费的算力，亦显学术严谨）。
